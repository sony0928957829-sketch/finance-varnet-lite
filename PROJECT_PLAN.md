# VARnet-lite 協作開發計畫 v1.0

> 本文件是人類（林）、Claude、Codex 三方的共同路線圖。
> 任何 agent 修改開發方向前，必須先修改本文件並經人類核准。
> 最後更新：2026-06-12

---

## 核心原則（不可違反）

1. **資料驅動，不憑空預測。** 所有預測輸出必須是可回溯到歷史資料的條件分布陳述
   （「此條件下歷史上 80% 落在此區間」），禁止任何 agent 產出點預測或買賣建議。
2. **機器裁判優先於 agent 判斷。** 模型改動的好壞由 `evaluate_range_coverage` 的
   校準度與區間寬度決定，不由任何 agent 的主觀評估決定。
3. **嚴格因果。** 任何特徵在 t 日的值只能使用 t 日（含）之前、且在該標的收盤時刻
   已實際存在的資料。跨市場 merge 必須先聲明兩個序列的實際成交時間順序。
4. **main 受保護。** 所有變更走 PR，合併權限只在人類手上。
5. **資料優先於模型。** 在窮盡更好的條件化資料之前，不引入更複雜的模型。

## 角色分工

| 角色 | 負責範圍 | 不負責 |
|---|---|---|
| 人類（林） | 方向決策、PR 合併、資料源帳號申請、實盤判斷 | 日常程式碼撰寫 |
| Claude | 模型設計、回測方法論、因果性審查、審查 Codex 的模型相關 PR | 大量 fetcher 工程 |
| Codex | Fetcher 接入、CI 維護、報告生成、資料管線工程 | 修改評估協議、修改模型邏輯（需 Claude 審查） |

衝突解決：兩個 agent 對同一問題給出不同方案時，由覆蓋率回測數字裁決；
數字無法裁決的（架構選擇等），由人類裁決。

## Phase 1：評估基準凍結（預計 3–5 天）

**目標：建立 v0.2 之後所有模型必須打敗的官方基線數字。**

- [ ] 1.1 用 yfinance 抓滿 5 年歷史，存成版本化的基準資料集
      （`data/benchmark/` + 資料雜湊記錄，確保未來比較基於同一份資料）
- [ ] 1.2 跑無條件基線（現行 252d 經驗分位數）的完整覆蓋率報告：
      按標的 × 按 horizon × 按波動率三分位分組
- [ ] 1.3 跑 ATR 條件化版本（`volatility_conditioning=True`），與 1.2 對比
- [ ] 1.4 將兩份結果寫入 `BENCHMARKS.md`，凍結為官方基線
- [ ] 1.5 Claude 審查：確認基準資料集無存活者偏差、無時區錯位

**驗收：** `BENCHMARKS.md` 存在，數字可由任何人重跑復現。
**負責：** Claude 設計協議，Codex 實作腳本，人類核准凍結。

## Phase 2：台股結構性資料接入（預計 2–3 週，與 Phase 1 部分並行）

**目標：取得價格序列之外的條件資訊——這是「資料整理驅動預測」的核心。**

優先順序（按預期資訊量 ÷ 接入成本排序）：

- [ ] 2.1 三大法人買賣超（TWSE/TPEx 公開 API）— Codex
- [ ] 2.2 融資融券餘額 — Codex
- [ ] 2.3 台指期三大法人未平倉、外資期貨淨部位（期交所）— Codex
- [ ] 2.4 選擇權 Put/Call ratio — Codex
- [ ] 2.5 每項資料完成後：Claude 審查發布時間（公布於收盤後幾點？
      → 決定可用於 t 日還是 t+1 日特徵），寫入 `DATA_ARCHITECTURE.md`
- [ ] 2.6 資料健康檢查擴充：每個新資料源納入 data_health 管線

**因果性紅線：** 三大法人資料約 15:00 後公布、期交所部位資料約 15:00–16:00 公布
——全部只能當作 **t+1 日**的特徵。每個 fetcher 的 docstring 必須註明公布時刻。

**驗收：** 每個資料源有 fetcher + normalizer + 健康檢查 + 公布時刻文件 + 測試。

## Phase 3：條件化區間模型 v0.2（預計 2–4 週）

**目標：在維持校準（±0.05 內）的前提下，把預測區間做窄。**

- [ ] 3.1 條件分組分位數：按波動率 regime × 法人籌碼狀態分組取經驗分位數
      （仍是「資料整理」，零模型假設）— Claude 設計，Codex 實作
- [ ] 3.2 分位數回歸（quantile regression / gradient boosting with pinball loss）：
      輸入現有特徵 + Phase 2 籌碼特徵，直接輸出條件分位數 — Claude
- [ ] 3.3 每個候選模型：walk-forward 驗證（訓練窗滾動，嚴禁全期重訓回看），
      與 BENCHMARKS.md 對比，結果進 PR comment
- [ ] 3.4 模型選擇規則：校準合格者中取平均區間最窄者；皆不合格則保留基線
- [ ] 3.5 報告模板更新：區間預測標注所用模型版本與當期校準狀態

**明確排除：** LSTM/Transformer 等深度模型在此階段不引入——9 檔標的 × 5 年
日線的樣本量不足以支撐，引入只會過擬合。樣本或標的池擴大後再議。

**驗收：** v0.2 模型在凍結基準上，至少一個 horizon 的平均區間寬度
比基線窄 ≥10% 且單邊覆蓋率在 0.75–0.85 之間。達不到就誠實保留基線。

## Phase 4：觸價機率 v0.3（待 Phase 3 驗收後規劃細節）

- 從區間分布自然延伸：P(h 日內觸及 ±x%) 是同一條件分布的另一種切面
- 驗收標準預先聲明:Brier score 與可靠度圖（reliability diagram），
  對比無條件歷史頻率基線

## Phase 5：策略回測與 paper trading v0.4（待 v0.3 後）

- 回測引擎需處理：交易成本、台股漲跌停、流動性、訊號延遲（t 日訊號 t+1 開盤執行）
- Paper trading 至少 3 個月，且其間禁止根據 paper 結果回頭調參（防止隱性過擬合）
- **此階段輸出仍然不是投資建議**；任何實盤決策由人類獨立做成

- 平日：CI 自動跑日報，異常開 GitHub Issue
- 週末：人類 review 本週 PR 與覆蓋率趨勢，決定下週優先級
- 每階段結束：Claude 出一份方法論回顧（什麼有效、什麼該砍）

## 風險登記簿

| 風險 | 緩解 |
|---|---|
| 兩個 agent 假設不一致互相覆寫 | 共同規範文件 + PR 審查分工 + 人類合併權 |
| 免費資料源（yfinance）斷供或改格式 | data_health 管線 + fallback 路由（已有） |
| 過擬合：模型在回測好、實際差 | 凍結基準 + walk-forward + paper trading 禁調參 |
| 範圍蔓延：追新模型而非資料品質 | 核心原則第 5 條，模型 PR 必須引用其使用的新資料 |
